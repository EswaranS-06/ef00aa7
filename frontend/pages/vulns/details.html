SQL Injection in Login

Critical CVSS 9.8 | OWASP A03:2021 Injection

The application is vulnerable to SQL Injection via the 'username' parameter in the '/api/login' endpoint. The input is not properly sanitized before being used in a SQL query, allowing an attacker to manipulate the query structure.

  1. Navigate to the Login Page.
  2. Enter the payload ' OR '1'='1 in the Username field.
  3. Enter any password.
  4. Observe that you are logged in as Administrator.

An attacker can bypass authentication, access sensitive user data, modify database records, or potentially gain full administrative access to the application.

Use Parameterized Queries (Prepared Statements) for all database access. Ensure that input validation is performed on the server side.
